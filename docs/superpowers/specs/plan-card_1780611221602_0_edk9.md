## Architecture Decisions

### 1. **Modular File Organization**
- **Rationale:** Separates concerns (app UI, scraping logic, session management, data) to enable independent testing and future refactoring. Each module has a clear responsibility.
- **Tradeoff:** Adds 4 files instead of monolithic `main.py`, but enables better code organization and testability.

### 2. **Virtual Environment Isolation**
- **Rationale:** Prevents dependency conflicts with system Python. Enables reproducible builds across environments.
- **Implementation:** Venv standard library (no additional tooling overhead).

### 3. **Ubuntu 26.04 Playwright Compatibility**
- **Rationale:** Ubuntu 26.04 may report itself as incompatible with Playwright's prebuilt binaries. Explicit `PLAYWRIGHT_HOST_PLATFORM_OVERRIDE=ubuntu24.04-x64` maps to compatible upstream binary release.
- **Tradeoff:** Hardcodes Ubuntu 24.04 x64 assumption. For other architectures (ARM, WSL2 native), this must be adjusted.
- **When to change:** If deployment target differs, update override in `.env` or build script.

### 4. **Security-First `.gitignore`**
- **Rationale:** LinkedIn scraping involves auth tokens, session cookies, and sensitive data. Explicit exclusions prevent accidental commits:
  - `*.db` / `*.json` — Database files and potential token dumps
  - `venv/` — Local env (can contain credentials in .pth files)
  - `__pycache__/`, `*.pyc` — Build artifacts
  - `.env` — Runtime secrets
- **Implementation:** Blocklist approach (explicit denials) rather than allowlist (safer for security-sensitive code).

### 5. **Requirements Pinning Strategy**
- **Rationale:** Exact version list (no `>=` or `~=`) ensures reproducible installs. Critical for Playwright which requires matching browser binary versions.
- **Packages:**
  - `playwright` — Browser automation
  - `playwright-stealth` — Anti-bot detection evasion
  - `beautifulsoup4` — HTML parsing
  - `pandas` — Data manipulation (CSV export, filtering)
  - `pydantic` — Data validation (config, API responses)
  - `streamlit` — Web UI framework (output/dashboard)

### 6. **Two-Stage Browser Setup**
- **Rationale:** `playwright install chromium` pulls browser binaries; `playwright install-deps` installs native OS dependencies (Xvfb, libgconf, fonts). Both required on headless systems.
- **Separation:** Enables CI/CD to cache browser layer separately from OS deps.

```python
#!/usr/bin/env python3
"""
Verify LinkedIn scraper project setup.
Checks Python version, dependencies, and Playwright browser availability.
"""
import sys
import subprocess

def check_python_version():
    """Verify Python 3.8+"""
    if sys.version_info < (3, 8):
        print(f"❌ Python {sys.version_info.major}.{sys.version_info.minor} < 3.8")
        return False
    print(f"✅ Python {sys.version_info.major}.{sys.version_info.minor}.{sys.version_info.micro}")
    return True

def check_imports():
    """Verify core dependencies are installed"""
    packages = [
        'playwright',
        'playwright_stealth',
        'bs4',
        'pandas',
        'pydantic',
        'streamlit',
    ]
    all_ok = True
    for pkg in packages:
        try:
            __import__(pkg)
            print(f"✅ {pkg}")
        except ImportError:
            print(f"❌ {pkg} not found")
            all_ok = False
    return all_ok

def check_playwright_browser():
    """Verify Chromium browser is installed"""
    try:
        result = subprocess.run(
            ['python', '-m', 'playwright', 'install', '--with-deps', 'chromium', '--dry-run'],
            capture_output=True,
            timeout=30
        )
        if result.returncode == 0:
            print("✅ Playwright Chromium browser available")
            return True
        else:
            print("❌ Playwright Chromium browser not found or install failed")
            return False
    except Exception as e:
        print(f"❌ Could not verify Chromium: {e}")
        return False

def main():
    print("=" * 50)
    print("LinkedIn Scraper Setup Verification")
    print("=" * 50)
    
    checks = [
        ("Python Version", check_python_version),
        ("Dependencies", check_imports),
        ("Playwright Browser", check_playwright_browser),
    ]
    
    results = []
    for name, check_fn in checks:
        print(f"\n{name}:")
        results.append(check_fn())
    
    print("\n" + "=" * 50)
    if all(results):
        print("✅ Setup verification PASSED")
        return 0
    else:
        print("❌ Setup verification FAILED")
        print("Run: pip install -r requirements.txt")
        print("Then: python -m playwright install chromium")
        return 1

if __name__ == '__main__':
    sys.exit(main())
```

## State Management Strategy

### Environment Variables
- **Runtime:** `PLAYWRIGHT_HOST_PLATFORM_OVERRIDE=ubuntu24.04-x64` (session-scoped during installation; persists in `.env` for later use)
- **Credentials:** Stored in `.env` (git-ignored), never committed
- **Playwright cache:** `~/.cache/ms-playwright/` (user home, system-managed)

### Virtual Environment State
- **Location:** `venv/` (git-ignored, local to project)
- **Activation:** Must be activated before running `pip install` or application
- **Isolation:** Python, pip, and all packages isolated from system Python

### Git Tracking
- **Tracked:** Source files, configuration, documentation (`.py`, `.txt`, `.md`, `.gitignore`)
- **Ignored:** Virtual environment, build artifacts, secrets, compiled Python (`__pycache__/`, `*.pyc`)
- **Commitment Strategy:** Only configuration and source files committed; dependencies managed via `requirements.txt`

## Risks and Edge Cases

### Risk 1: Ubuntu 26.04 Platform Override Doesn't Match
**Symptom:** Playwright fails to find compatible browser binary; error like "chromium not found for linux" or version mismatch
**Cause:** `PLAYWRIGHT_HOST_PLATFORM_OVERRIDE=ubuntu24.04-x64` assumes x64 architecture and Ubuntu 24.04 binary compatibility
**Mitigation:**
- For ARM64 systems: Change to `ubuntu24.04-arm64`
- For non-Ubuntu Debian: Override may still work; test with `python -m playwright install chromium --with-deps` (ignoring override first)
- Document in README which OS versions tested

### Risk 2: Playwright Browser Installation Requires sudo for OS Dependencies
**Symptom:** `playwright install-deps` prompts for password or fails with permission error
**Cause:** Installing system packages (libxkbcommon, fonts) requires elevated privileges on some systems
**Mitigation:**
- Expect sudo prompt; document in README
- For CI/CD, pre-install dependencies via Dockerfile or system package manager
- For local development, allow sudo for pip/playwright commands

### Risk 3: Network Interruption During Browser Download
**Symptom:** `playwright install chromium` fails halfway through; incomplete binary
**Cause:** Chromium binary is 150MB+; unreliable networks may drop connection
**Mitigation:**
- Retry with `playwright install chromium` (idempotent; resumes download)
- For flaky networks, consider pre-built Docker image with browser baked in
- Document in troubleshooting section of README

### Risk 4: Virtual Environment Not Activated Before pip install
**Symptom:** Dependencies installed globally to system Python instead of venv
**Cause:** User forgets `source venv/bin/activate` step
**Mitigation:**
- Clearly highlight activation step in README
- Add shell prompt check in setup script (detect `(venv)` prefix)
- Provide .bashrc / .zshrc snippet for automatic activation

### Risk 5: .env File Accidentally Committed
**Symptom:** LinkedIn credentials exposed in Git history
**Cause:** `.env` not in `.gitignore` or developer adds it explicitly
**Mitigation:**
- Explicit `.gitignore` rule for `.env` (included in this plan)
- Provide `.env.example` as template; document "never commit .env"
- Pre-commit hook (future task) could block `.env` commits

### Risk 6: Playwright Stealth Plugin Doesn't Prevent LinkedIn Bot Detection
**Symptom:** LinkedIn detects automated browser; blocks scraping or requires CAPTCHA
**Cause:** LinkedIn has sophisticated bot detection; stealth plugin may not be sufficient
**Mitigation:**
- This is expected for LinkedIn scraping; stealth helps but is not foolproof
- Document this limitation in README
- Recommend rate limiting, realistic User-Agent, delays between requests (future scraper.py tasks)
- Consider using Playwright's `launch_persistent_context` for cookie-based auth

### Risk 7: Python 3.7 or Earlier on System
**Symptom:** `python -m venv venv` fails or setup script errors on type hints (e.g., `list[str]`)
**Cause:** venv requires Python 3.3+; type hints `list[str]` require 3.9+
**Mitigation:**
- Verification script checks Python 3.8+ (supports venv, typing annotations)
- Document Python 3.8+ requirement in README
- For older systems, users must install Python 3.8+ or use pyenv

### Risk 8: .gitignore Patterns Too Permissive
**Symptom:** Sensitive files (*.json with tokens) accidentally committed
**Cause:** `.gitignore` patterns don't cover all cases (e.g., `config.json` but not `app-config.json`)
**Mitigation:**
- Explicit blocklist in plan: `*.db`, `*.json`, `.env`, `__pycache__/`
- Document sensitive file patterns in README
- Use `git check-ignore` to verify patterns work before committing real data

### Edge Case 1: Windows or macOS Users
**Symptom:** `source venv/bin/activate` fails; command not found
**Cause:** Windows uses `venv\Scripts\activate.bat` or `venv\Scripts\Activate.ps1` (PowerShell)
**Mitigation:**
- Document platform-specific activation commands in README
- Future Windows testing; for now, Linux/Ubuntu focus is acceptable

### Edge Case 2: Multiple Python Versions on System
**Symptom:** `python` resolves to Python 2.7 or 3.6; venv creation fails
**Cause:** System has multiple Python versions; `python` is not Python 3.8+
**Mitigation:**
- Use explicit `python3` or `python3.10` (check system)
- Verification script documents which `python` is active
- README recommends `python3 -m venv venv` (explicit)

### Edge Case 3: Git Not Installed or Configured
**Symptom:** `git init` fails or "author identity unknown" error on commit
**Cause:** Git not installed or user.name/user.email not configured
**Mitigation:**
- Assume Git is installed (common on dev systems)
- Verification script does not check Git (not a Python dependency)
- Document in README if issues arise
